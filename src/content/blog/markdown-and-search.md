写博客最怕的不是没灵感，而是内容分散在太多地方。

所以我更喜欢下面这套方式：

1. 文章正文就是 `.md` 或 `.mdx`
2. 网站负责展示、分类和检索
3. 图片、插图和公式直接跟着文章一起管理

这样做有几个好处：

| 能力 | 作用 |
| --- | --- |
| Markdown 写作 | 可以用任何熟悉的编辑器继续写 |
| 分类归档 | 方便把文章组织成长期专题 |
| 全文检索 | 站点上线后依然可以快速回查 |
| 版本管理 | 用 Git 追踪所有改动 |

后续我只要继续往 `src/content/blog` 里增加文章，这个站点就会自动长出新的页面。
