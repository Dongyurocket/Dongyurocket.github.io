这篇示例笔记的目的不是讲某一个具体学科，而是示范：**如果你以后要写一篇结构清晰、方便长期回查的研究型笔记，可以怎么组织。**

你可以直接把这篇当作模板，替换成自己的内容。

## 1. 研究问题

用一段话或一个引用块说清楚你到底要解决什么问题：

> 为"长期写作系统"设计一套统一的笔记结构，让一篇笔记既能在网站中良好展示，也能顺畅同步到微信公众号。

然后列出需要回答的关键问题：

1. 我到底要解决什么问题？
2. 我判断有效的标准是什么？
3. 我需要记录哪些中间过程？
4. 最后怎样把内容转化成可执行的下一步？

## 2. 结构图

![研究笔记工作流示意图](./rich-notes-example-files/notebook-workflow.svg)

一篇研究笔记通常包含六个核心区块：

- 问题定义
- 资料整理
- 实验设计
- 结果记录
- 分析与公式
- 结论与行动

## 3. 记录框架

下面这张表可以作为每篇笔记的最小骨架：

| 模块 | 建议记录内容 | 为什么重要 |
| --- | --- | --- |
| 问题定义 | 目标、约束、假设 | 防止写着写着偏题 |
| 背景资料 | 参考文献、数据来源、前置结论 | 便于后续追溯 |
| 实验设计 | 变量、步骤、评价指标 | 保证过程可复现 |
| 结果记录 | 表格、现象、截图、异常 | 防止只记结论不记过程 |
| 分析推导 | 公式、模型、误差分析 | 把"感觉"变成"证据" |
| 行动项 | 结论、问题、下一步 | 让笔记真正推动行动 |

## 4. 公式示例

如果需要数学表达，可以直接用 LaTeX。行内公式如 $S = \alpha C + \beta E$，独立公式块：

$$
S = \alpha C + \beta E + \gamma R + \delta A
$$

其中：

- $C$ 表示内容完整性，$E$ 表示证据充分性
- $R$ 表示可复现程度，$A$ 表示行动可执行性
- 约束：$\alpha + \beta + \gamma + \delta = 1$

## 5. 实验记录表

| 方案 | 完整性 C | 可复现 R | 同步难度 | 总体评价 |
| --- | ---: | ---: | ---: | --- |
| 纯 Markdown 结构化笔记 | 8.5 | 9.0 | 低 | 最推荐 |
| MDX + 多组件增强 | 9.5 | 7.0 | 高 | 适合网站专用 |
| 只写零散段落 | 4.0 | 3.5 | 中 | 不适合长期积累 |

> 如果追求长期写作与多端同步的平衡，结构化的纯 Markdown 往往是最优解。

## 6. 代码块

```python
def note_score(completeness, evidence, reproducibility, actionability):
    alpha, beta, gamma, delta = 0.30, 0.25, 0.25, 0.20
    return alpha * completeness + beta * evidence + gamma * reproducibility + delta * actionability
```

## 7. 检查清单

写完一篇研究型笔记后，可以用这个 checklist 快速自查：

- [ ] 问题是否定义清楚
- [ ] 是否交代了背景与假设
- [ ] 是否有公式或模型支撑分析
- [ ] 是否有表格或结果记录
- [ ] 是否有图片或结构图帮助理解
- [ ] 是否有清晰结论
- [ ] 是否写出了下一步行动

## 8. 总结

> 一篇真正有长期价值的笔记，不只是记录结论，而是记录"问题—过程—证据—结论—行动"这一整条链路。
